# Regras para TypeScript

- strict mode sempre
- Interfaces > types (exceto unions)
- Componentes React como function components com arrow functions
- Tailwind CSS para estilização
- Zod para validação de schemas
- Server Components por padrão (Next.js App Router)
- Tratar loading, error e empty states em toda UI
