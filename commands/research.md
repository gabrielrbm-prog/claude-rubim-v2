Faça uma **pesquisa profunda** sobre: $ARGUMENTS

## Processo:

### Fase 1: Coleta (paralela)
Lance 3 agentes em paralelo:
1. **Web Search** — busque nas principais fontes (blogs, docs oficiais, papers)
2. **GitHub Search** — busque repos, issues, discussions relevantes
3. **Documentation** — busque docs oficiais e exemplos de código

### Fase 2: Análise
- Cruze informações das 3 fontes
- Identifique consensos e divergências
- Priorize fontes primárias (docs oficiais > blogs > opiniões)

### Fase 3: Síntese
Apresente um relatório estruturado:

## Relatório
1. **Resumo executivo** (3-5 frases)
2. **Estado atual** — o que existe hoje
3. **Opções disponíveis** — tabela comparativa
4. **Recomendação** — baseada no contexto do projeto
5. **Próximos passos** — ações concretas
6. **Fontes** — links para referências

## Regras:
- Sempre cite as fontes
- Distingua fatos de opiniões
- Priorize informação recente (< 6 meses)
- Se encontrar conflito entre fontes, destaque
