Implemente a feature full-stack: $ARGUMENTS

## Fluxo de implementação:

### 1. Análise (pesquise antes de codar)
- Entenda os requisitos da feature
- Verifique código existente que pode ser reutilizado
- Identifique APIs/serviços necessários

### 2. Backend
- Models/schemas de dados (Pydantic/Zod)
- API endpoints com validação
- Service layer com lógica de negócio
- Error handling e logging
- Testes unitários

### 3. Frontend
- Componentes React/Next.js com TypeScript
- Integração com APIs do backend
- Estados: loading, error, empty, success
- Responsivo (mobile-first)
- Acessibilidade básica

### 4. Validação
- Rode os testes
- Verifique tipagem
- Faça um review rápido do código gerado

## Stack: Next.js + TypeScript + Tailwind (front) | Python + FastAPI (back) | PostgreSQL (banco)
